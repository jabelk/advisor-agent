# Implementation Plan: Market Data Integration

**Branch**: `003-market-data` | **Date**: 2026-02-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-market-data/spec.md`

## Summary

Add Alpaca market data integration — historical OHLCV bars (daily + hourly), real-time price snapshots, and technical indicators (SMA-20, SMA-50, RSI-14, VWAP). Bars are stored in SQLite with incremental fetch logic. Indicators persist latest values per ticker for the decision engine. New `market` CLI command group for fetch, snapshot, status, and indicator operations.

## Technical Context

**Language/Version**: Python 3.12+ (existing project)
**Primary Dependencies**: alpaca-py >=0.43 (existing) — `StockHistoricalDataClient`, `StockBarsRequest`, `StockSnapshotRequest`
**Storage**: SQLite (extends existing DB with 3 new tables: `price_bar`, `technical_indicator`, `market_data_fetch`), schema version 2 → 3
**Testing**: pytest with mocked Alpaca client (unit), live API calls (integration)
**Target Platform**: macOS/Linux (Intel NUC), CLI application
**Project Type**: Single project — extends existing `src/finance_agent/` layout
**Performance Goals**: 2yr daily fetch <30s, incremental <5s/ticker, snapshot <2s, indicator compute <1s/ticker
**Constraints**: Alpaca free-tier rate limit 200 req/min; IEX feed for real-time, SIP for historical (15min delay); split-adjusted prices only
**Scale/Scope**: Up to 20 watchlist companies, ~14K bars total, ~1.4 MB SQLite

**Structure Decision**: New `market/` module under `src/finance_agent/` — separate from `data/` (which handles research document ingestion) because market data serves the decision engine directly, not the LLM analysis pipeline.

## Key Design Decisions

1. **Not a BaseSource**: Market data is NOT an ingestion source — it doesn't produce documents for LLM analysis. It provides numerical data for the decision engine.
2. **Split-adjusted prices** (`Adjustment.SPLIT`): Prevents indicator corruption across stock splits.
3. **SIP for historical, IEX for real-time**: Free tier allows SIP historical if end date is 15+ min old.
4. **Rate limiter at 180 req/min**: 90% of free-tier limit (200) for safety margin.
5. **`INSERT OR IGNORE` dedup**: Composite unique constraint on `(ticker, timeframe, bar_timestamp)` handles incremental inserts efficiently.
6. **Pure Python indicators**: No numpy/pandas — SMA, RSI, VWAP are trivial calculations.
7. **Latest indicator values only**: Persisted for decision engine lookup, not full time series.
